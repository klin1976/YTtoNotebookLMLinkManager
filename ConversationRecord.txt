這是關於 Google Cloud Console 中 API 金鑰的安全性設定建議。

## 🔐 API 金鑰安全性設定說明

### 1. 應用程式限制（網站限制）
https://klin1976.github.io/*

這個設定的意思是：
- 只有來自 https://klin1976.github.io/ 網域的請求才能使用這個 API 金鑰
- * 是萬用字元，代表該網域下的所有路徑都可以使用
- 如果有人竊取了你的 API 金鑰，想從其他網站使用，Google 會拒絕該請求

### 2. API 限制（僅限 YouTube Data API v3）

這個設定的意思是：
- 這把金鑰只能用於呼叫 YouTube Data API v3
- 即使有人拿到你的金鑰，也無法用它來呼叫其他 Google API
- 這可以防止金鑰被濫用於其他用途

### 如何設定？
1. 前往 Google Cloud Console (https://console.cloud.google.com/)
2. 選擇你的專案
3. 左側選單 → API 和服務 → 憑證
4. 點擊你的 API 金鑰
5. 設定：
   - 應用程式限制 → 選擇「HTTP 參照網址（網站）」→ 新增 https://klin1976.github.io/*
   - API 限制 → 選擇「限制金鑰」→ 只勾選 YouTube Data API v3

### 為什麼這很重要？
| 風險           | 沒有限制             | 有限制           |
|----------------|---------------------|------------------|
| API 金鑰外洩   | 任何人都能用你的配額 | 只有你的網站能用 |
| 被惡意使用     | 可能產生大量費用     | 請求會被拒絕     |
| 濫用其他 API   | 金鑰可用於任何API    | 只能用於YouTube  |

這是最佳實踐，強烈建議設定這些限制！
